# 桂林字牌项目完善说明文档

## 1. 核心规则更新 (Core Logic)
当前版本已严格对齐《桂林字牌深度调研报告》中的地方规则：
*   **胡息系统**：
    *   **吃牌 (Chi)**：仅 `1-2-3` 和 `2-7-10` 组合产生胡息（大牌 6，小牌 3）。
    *   **碰牌 (Peng)**：大牌 3，小牌 1。
    *   **坎/扫 (Kan/Sao)**：大牌 6，小牌 3。
    *   **扫穿/开砣 (SaoChuan/KaiDuo)**：大牌 12，小牌 9。
*   **胡牌判定**：最低胡息阈值设为 **10息**。

## 2. 经济系统 (Economy System)
实现了完整的闭环对局收益逻辑：
*   **自动结算**：对局结束时自动计算输赢豆、扣除系统抽水 (Rake: 5%)。
*   **段位与积分**：集成 Elo 积分系统和星级进阶，包含从“青铜”到“王者”的 7 个段位。
*   **福利机制**：包含每日免费领取欢乐豆和破产补助（每日限领3次）。

## 3. 技能系统 (Skill System)
在基础技能上增加了资源管理属性，增强了对局的策略深度：
*   **能量管理 (SP)**：玩家初始 100 SP。强力技能需消耗能量（如：撤回出牌 50 SP）。
*   **冷却管理 (CD)**：技能使用后进入转冷周期，防止连续滥用。
*   **技能实例**：包含信息类（听势、算余）、容错类（稳手、重整）、收益类（稳豆、加码）等 12 种技能。

## 4. 服务器集成 (Server Integration)
*   **连接管理**：WebSocket 连接时自动初始化玩家经济账户。
*   **结算同步**：在有人胡牌后，服务器自动执行 `check_game_over` 并广播最终的欢乐豆及积分变动。

# 开发运行指南

### 环境需求
*   **Rust**: 1.75+
*   **Trunk**: `cargo install trunk` (用于前端 WASM 构建)
*   **WASM Target**: `rustup target add wasm32-unknown-unknown`

### 启动方式
1.  **运行服务器**：
    ```powershell
    cd crates/server
    cargo run --release
    ```
2.  **运行客户端**：
    ```powershell
    # 打开新窗口
    trunk serve
    ```
3.  **访问项目**：
    在浏览器打开 `http://127.0.0.1:3000` (Trunk 默认端口) 或 `8080`。

### 系统测试
*   **全项目自测**：`cargo test --workspace`
*   **功能模块校验**：
    *   规则校验：`cargo test -p guilin-paizi-core`
    *   经济校验：`cargo test -p guilin-paizi-economy`
